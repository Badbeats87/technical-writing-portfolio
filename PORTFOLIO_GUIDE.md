# How to Use This Portfolio

This portfolio is organized to showcase different technical writing skills and styles. Here's how to present it effectively.

## Quick Start

1. **Start with the README.md** - This gives an overview of your writing and the organization
2. **Browse by Category** - Each numbered folder represents a different writing style:
   - Read the INDEX.md in each folder for context about those pieces
   - Then review the actual documentation files

3. **Share Selectively** - Depending on the role, you might emphasize different categories

## How to Present to Hiring Managers

### For Technical Writer Roles
Emphasize:
- **All categories** - Shows breadth of documentation types
- **Feature Documentation (Category 3)** - Demonstrates complex technical understanding
- **Reference Material (Category 4)** - Shows ability to explain abstract concepts clearly

### For Product Manager Roles
Emphasize:
- **Best Practices (Category 2)** - Shows user-centric thinking
- **Feature Documentation (Category 3)** - Demonstrates product knowledge

### For DevOps/Cloud Documentation Roles
Emphasize:
- **Troubleshooting and Support (Category 5)** - Problem-solving documentation
- **Instructional Guides (Category 1)** - Setup and configuration guidance
- **Reference Material (Category 4)** - System architecture understanding

### For UX/Content Design Roles
Emphasize:
- **Instructional Guides (Category 1)** - User-centric language and accessibility
- **Best Practices (Category 2)** - User journey and decision-support thinking
- **Reference Material (Category 4)** - Information architecture and organization

## Common Talking Points

**Scenario 1: "Tell me about a complex feature you documented"**
→ Reference the Intraday Adherence or Real-Time Adherence articles. Discuss how you made metrics, calculations, and status indicators clear to users.

**Scenario 2: "How do you handle different user skill levels?"**
→ Reference the Instructional Guides and how language, prerequisites, and verification are tailored to end-users versus administrators.

**Scenario 3: "How do you approach troubleshooting documentation?"**
→ Reference the Resolve Scheduling Errors article and discuss the systematic approach to diagnosing and solving problems.

**Scenario 4: "Can you show us your best work?"**
→ The Best Practices guides showcase nuanced thinking about real-world scenarios, trade-offs, and multiple valid approaches—often the most valuable documentation.
