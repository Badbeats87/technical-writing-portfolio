## Writing Approach

All feature documentation follows these principles:
- **Comprehensive Scope:** Overview + detailed information
- **Multiple Perspectives:** Admin vs. end-user, different use cases
- **Clear Prerequisites:** What's needed before using the feature
- **Configuration Details:** How to set up and configure
- **Practical Examples:** Real-world usage scenarios
- **Visual Organization:** Tables, lists, and formatting for clarity
- **Reference Quality:** Can be used as a lookup resource
- **Troubleshooting:** Common issues and how to resolve them
